# Observer Pattern
Imagine there is an object which produces data. A lot of other components are dependent on this data and need to keep track of the data that is generated by the subject. 

The objects that need to track the changes of the data generating object are called observers and the data generating object is called the subject.

### Key Principle
Instead of the observers constantly pinging the subject for latest data, the subject notifies all the observers whenever there is any change to its data.

The observers register themselves (subscribe) with the subject that they need to track and the subject then the subject notifies all its observers (publish). Each observer has its own functionality apart from being an observer. The same is true of the subject as well.

A real world example is when people subscribe to a magazine. Whenever there is a new edition, the magazine is delivered to all the subscribed people. The people who are subscribed are free to unsubscribe at any point in time. Post that, they would not receive the latest editions anymore.
